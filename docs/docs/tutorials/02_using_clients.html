Using SmartRedis Clients¶

2.1 Python Client¶

In this section, examples are presented using the SmartRedis Python API to interact with the RedisAI tensor, model, and script data types. Additionally, an example of utilizing the SmartRedis DataSet API is also provided.

Note

The Python API examples are written to connect to a database at 127.0.0.1:6379. When running this example, ensure that the address and port of your Redis instance are used.

Note

The Python API examples are written to connect to a non-cluster Redis database. Update the Client constructor call to connect to a Redis cluster.

Tensors¶

The Python client has the ability to send and receive tensors from the Redis database. The tensors are stored in the Redis database as RedisAI data structures. Additionally, Python client API functions involving tensor data are compatible with Numpy arrays and do not require any other data types.

 1import numpy as np
 2from smartredis import Client
 3
 4# Connect a SmartRedis client to Redis database
 5db_address = "127.0.0.1:6379"
 6client = Client(address=db_address, cluster=True)
 7
 8# Send a 2D tensor to the database
 9key = "2D_array"
10array = np.random.randint(-10, 10, size=(10, 10))
11client.put_tensor(key, array)
12
13# Retrieve the tensor
14returned_array = client.get_tensor("2D_array")

Datasets¶

The Python client can store and retrieve tensors and metadata in datasets. For further information about datasets, please refer to the Dataset section of the Data Structures documentation page.

The code below shows how to store and retrieve tensors which belong to a DataSet.

 1import numpy as np
 2
 3from smartredis import Client, Dataset
 4
 5# Create two arrays to store in the DataSet
 6data_1 = np.random.randint(-10, 10, size=(10,10))
 7data_2 = np.random.randint(-10, 10, size=(20, 8, 2))
 8
 9# Create a DataSet object and add the two sample tensors
10dataset = Dataset("test-dataset")
11dataset.add_tensor("tensor_1", data_1)
12dataset.add_tensor("tensor_2", data_2)
13
14# Connect SmartRedis client to Redis database
15db_address = "127.0.0.1:6379"
16client = Client(address=db_address, cluster=True)
17
18# Place the DataSet into the database
19client.put_dataset(dataset)
20
21# Retrieve the DataSet from the database
22rdataset = client.get_dataset("test-dataset")
23
24# Retrieve a tensor from inside of the fetched
25# DataSet
26rdata_1 = rdataset.get_tensor("tensor_1")

Models¶

The SmartRedis clients allow users to set and use a PyTorch, ONNX, TensorFlow, or TensorFlow Lite model in the database. Models can be sent to the database directly from memory or from a file. The code below illustrates how a jit-traced PyTorch model can be used with the Python client library.

 1import io
 2
 3import torch
 4import torch.nn as nn
 5
 6from smartredis import Client
 7
 8# Taken from https://pytorch.org/docs/master/generated/torch.jit.trace.html
 9class Net(nn.Module):
10    def __init__(self):
11        super(Net, self).__init__()
12        self.conv = nn.Conv2d(1, 1, 3)
13
14    def forward(self, x):
15        return self.conv(x)
16
17net = Net()
18example_forward_input = torch.rand(1, 1, 3, 3)
19
20# Trace a module (implicitly traces `forward`) and construct a
21# `ScriptModule` with a single `forward` method
22module = torch.jit.trace(net, example_forward_input)
23
24# Create a buffer of the traced model
25buffer = io.BytesIO()
26torch.jit.save(module, buffer)
27model = buffer.getvalue()
28
29# Connect a SmartRedis client and set the model in the database
30db_address = "127.0.0.1:6379"
31client = Client(address=db_address, cluster=True)
32client.set_model("torch_cnn", model, "TORCH", "CPU")
33
34# Retrieve the model and verify that the retrieved
35# model matches the original model.
36returned_model = client.get_model("torch_cnn")
37assert model == returned_model
38
39# Setup input tensor
40data = torch.rand(1, 1, 3, 3).numpy()
41client.put_tensor("torch_cnn_input", data)
42
43# Run model and get output
44client.run_model("torch_cnn", inputs=["torch_cnn_input"], outputs=["torch_cnn_output"])
45out_data = client.get_tensor("torch_cnn_output")

Models can also be set from a file, as in the code below.

 1import os
 2
 3import torch
 4import torch.nn as nn
 5
 6from smartredis import Client
 7
 8
 9# taken from https://pytorch.org/docs/master/generated/torch.jit.trace.html
10class Net(nn.Module):
11    def __init__(self):
12        super(Net, self).__init__()
13        self.conv = nn.Conv2d(1, 1, 3)
14
15    def forward(self, x):
16        return self.conv(x)
17
18# Connect a SmartRedis client
19db_address = "127.0.0.1:6379"
20client = Client(address=db_address, cluster=True)
21
22try:
23    net = Net()
24    example_forward_input = torch.rand(1, 1, 3, 3)
25    # Trace a module (implicitly traces `forward`) and construct a
26    # `ScriptModule` with a single `forward` method
27    module = torch.jit.trace(net, example_forward_input)
28
29    # Save the traced model to a file
30    torch.jit.save(module, "./torch_cnn.pt")
31
32    # Set the model in the Redis database from the file
33    client.set_model_from_file("file_cnn", "./torch_cnn.pt", "TORCH", "CPU")
34
35    # Put a tensor in the database as a test input
36    data = torch.rand(1, 1, 3, 3).numpy()
37    client.put_tensor("torch_cnn_input", data)
38
39    # Run model and retrieve the output
40    client.run_model("file_cnn", inputs=["torch_cnn_input"], outputs=["torch_cnn_output"])
41    out_data = client.get_tensor("torch_cnn_output")
42finally:
43    os.remove("torch_cnn.pt")

Scripts¶

Scripts are a way to store python-executable code in the database. The Python client can send scripts to the dataset from a file, or directly from memory.

As an example, the code below illustrates how a function can be defined and sent to the database on the fly, without storing it in an intermediate file.

 1import numpy as np
 2import torch
 3
 4from smartredis import Client
 5
 6def two_to_one(data, data_2):
 7    """Sample torchscript script that returns the
 8    highest elements in both arguments
 9
10    Two inputs to one output
11    """
12    # return the highest element
13    merged = torch.cat((data, data_2))
14    return merged.max(1)[0]
15
16# Connect a SmartRedis client to the Redis database
17db_address = "127.0.0.1:6379"
18client = Client(address=db_address, cluster=True)
19
20# Generate some test data to feed to the two_to_one function
21data = np.array([[1, 2, 3, 4]])
22data_2 = np.array([[5, 6, 7, 8]])
23
24# Put the test data into the Redis database
25client.put_tensor("script-data-1", data)
26client.put_tensor("script-data-2", data_2)
27
28# Put the function into the Redis database
29client.set_function("two-to-one", two_to_one)
30
31# Run the script using the test data
32client.run_script(
33    "two-to-one",
34    "two_to_one",
35    ["script-data-1", "script-data-2"],
36    ["script-multi-out-output"],
37)
38
39# Retrieve the output of the test function
40out = client.get_tensor("script-multi-out-output")

The code below shows how to set a script from a file. Running the script set from file uses the same API calls as the example shown above.

 1import os.path as osp
 2from smartredis import Client
 3
 4# Construct a string holding the script file location
 5file_path = osp.dirname(osp.abspath(__file__))
 6script_path = osp.join(file_path, "./data_processing_script.txt")
 7
 8# Connect to the Redis database
 9db_address = "127.0.0.1:6379"
10client = Client(address=db_address, cluster=True)
11
12# Place the script in the database
13client.set_script_from_file(
14    "test-script-file", osp.join(file_path, "./data_processing_script.txt")
15)

The content of the script file has to be written in Python. For the example above, the file data_processing_script.txt looks like this:

1def pre_process(inp):
2    mean = torch.zeros(1).float().to(inp.device)
3    mean[0] = 2.0
4    temp = inp.float() * mean
5    return temp

2.2 C++ Client¶

In this section, examples are presented using the SmartRedis C++ API to interact with the RedisAI tensor, model, and script data types. Additionally, an example of utilizing the SmartRedis DataSet API is also provided.

Note

The C++ API examples rely on the SSDB environment variable being set to the address and port of the Redis database.

Note

The C++ API examples are written to connect to a non-cluster Redis database. Update the Client constructor call to connect to a Redis cluster.

Tensors¶

The following example shows how to send a receive a tensor using the SmartRedis C++ client API.

 1#include "client.h"
 2#include <vector>
 3#include <string>
 4
 5int main(int argc, char* argv[]) {
 6
 7    // Initialize tensor dimensions
 8    size_t dim1 = 3;
 9    size_t dim2 = 2;
10    size_t dim3 = 5;
11    std::vector<size_t> dims = {3, 2, 5};
12
13    // Initialize a tensor to random values.  Note that a dynamically
14    // allocated tensor via malloc is also useable with the client
15    // API.  The std::vector is used here for brevity.
16    size_t n_values = dim1 * dim2 * dim3;
17    std::vector<double> input_tensor(n_values, 0);
18    for(size_t i=0; i<n_values; i++)
19        input_tensor[i] = 2.0*rand()/RAND_MAX - 1.0;
20
21    // Initialize a SmartRedis client
22    bool cluster_mode = true; // Set to false if not using a clustered database
23    SmartRedis::Client client(cluster_mode);
24
25    // Put the tensor in the database
26    std::string key = "3d_tensor";
27    client.put_tensor(key, input_tensor.data(), dims,
28                      SRTensorTypeDouble, SRMemLayoutContiguous);
29
30    // Retrieve the tensor from the database using the unpack feature.
31    std::vector<double> unpack_tensor(n_values, 0);
32    client.unpack_tensor(key, unpack_tensor.data(), {n_values},
33                        SRTensorTypeDouble, SRMemLayoutContiguous);
34
35    // Print the values retrieved with the unpack feature
36    std::cout<<"Comparison of the sent and "\
37                "retrieved (via unpack) values: "<<std::endl;
38    for(size_t i=0; i<n_values; i++)
39        std::cout<<"Sent: "<<input_tensor[i]<<" "
40                 <<"Received: "<<unpack_tensor[i]<<std::endl;
41
42
43    // Retrieve the tensor from the database using the get feature.
44    SRTensorType get_type;
45    std::vector<size_t> get_dims;
46    void* get_tensor;
47    client.get_tensor(key, get_tensor, get_dims, get_type, SRMemLayoutNested);
48
49    // Print the values retrieved with the unpack feature
50    std::cout<<"Comparison of the sent and "\
51                "retrieved (via get) values: "<<std::endl;
52    for(size_t i=0, c=0; i<dims[0]; i++)
53        for(size_t j=0; j<dims[1]; j++)
54            for(size_t k=0; k<dims[2]; k++, c++) {
55                std::cout<<"Sent: "<<input_tensor[c]<<" "
56                         <<"Received: "
57                         <<((double***)get_tensor)[i][j][k]<<std::endl;
58    }
59
60    return 0;
61}

DataSets¶

The C++ client can store and retrieve tensors and metadata in datasets. For further information about datasets, please refer to the Dataset section of the Data Structures documentation page.

The code below shows how to store and retrieve tensors and metadata which belong to a DataSet.

 1#include "client.h"
 2#include <vector>
 3#include <string>
 4
 5int main(int argc, char* argv[]) {
 6
 7    // Initialize tensor dimensions
 8    size_t dim1 = 3;
 9    size_t dim2 = 2;
10    size_t dim3 = 5;
11    size_t n_values = dim1 * dim2 * dim3;
12    std::vector<size_t> dims = {3, 2, 5};
13
14    // Initialize two tensors to random values
15    std::vector<double> tensor_1(n_values, 0);
16    std::vector<int64_t> tensor_2(n_values, 0);
17
18    for(size_t i=0; i<n_values; i++) {
19        tensor_1[i] = 2.0*rand()/RAND_MAX - 1.0;
20        tensor_2[i] = rand();
21    }
22
23    // Initialize three metadata values we will add
24    // to the DataSet
25    uint32_t meta_scalar_1 = 1;
26    uint32_t meta_scalar_2 = 2;
27    int64_t meta_scalar_3 = 3;
28
29    // Initialize a SmartRedis client
30    bool cluster_mode = true; // Set to false if not using a clustered database
31    SmartRedis::Client client(cluster_mode);
32
33    // Create a DataSet
34    SmartRedis::DataSet dataset("example_dataset");
35
36    // Add tensors to the DataSet
37    dataset.add_tensor("tensor_1", tensor_1.data(), dims,
38                       SRTensorTypeDouble, SRMemLayoutContiguous);
39
40    dataset.add_tensor("tensor_2", tensor_2.data(), dims,
41                       SRTensorTypeInt64, SRMemLayoutContiguous);
42
43    // Add metadata scalar values to the DataSet
44    dataset.add_meta_scalar("meta_field_1", &meta_scalar_1, SRMetadataTypeUint32);
45    dataset.add_meta_scalar("meta_field_1", &meta_scalar_2, SRMetadataTypeUint32);
46    dataset.add_meta_scalar("meta_field_2", &meta_scalar_3, SRMetadataTypeInt64);
47
48
49    // Put the DataSet in the database
50    client.put_dataset(dataset);
51
52    // Retrieve the DataSet from the database
53    SmartRedis::DataSet retrieved_dataset =
54        client.get_dataset("example_dataset");
55
56    // Retrieve one of the tensors
57    std::vector<int64_t> unpack_dataset_tensor(n_values, 0);
58    retrieved_dataset.unpack_tensor("tensor_2",
59                                    unpack_dataset_tensor.data(),
60                                    {n_values},
61                                    SRTensorTypeInt64,
62                                    SRMemLayoutContiguous);
63
64    // Print out the retrieved values
65    std::cout<<"Comparing sent and received "\
66               "values for tensor_2: "<<std::endl;
67
68    for(size_t i=0; i<n_values; i++)
69        std::cout<<"Sent: "<<tensor_2[i]<<" "
70                 <<"Received: "
71                 <<unpack_dataset_tensor[i]<<std::endl;
72
73    //Retrieve a metadata field
74    size_t get_n_meta_values;
75    void* get_meta_values;
76    SRMetaDataType get_type;
77    dataset.get_meta_scalars("meta_field_1",
78                             get_meta_values,
79                             get_n_meta_values,
80                             get_type);
81
82    // Print out the metadata field values
83    for(size_t i=0; i<get_n_meta_values; i++)
84        std::cout<<"meta_field_1 value "<<i<<" = "
85                 <<((uint32_t*)get_meta_values)[i]<<std::endl;
86
87    return 0;
88}

Parallel (MPI) execution¶

In this example, the example shown in Models and Scripts is adapted to run in parallel using MPI. This example has the same functionality, however, it shows how keys can be prefixed to prevent key collisions across MPI ranks. Note that only one model and script are set, which is shared across all ranks.

For completeness, the pre-processing script source code is also shown.

C++ program

  1#include "client.h"
  2#include <mpi.h>
  3
  4void run_mnist(const std::string& model_name,
  5               const std::string& script_name,
  6               SmartRedis::Client& client)
  7{
  8    // Get the MPI rank
  9    int rank;
 10    MPI_Comm_rank(MPI_COMM_WORLD, &rank);
 11
 12    // Initialize a vector that will hold input image tensor
 13    size_t n_values = 1*1*28*28;
 14    std::vector<float> img(n_values, 0);
 15
 16    // Load the mnist image from a file using MPI rank 0
 17    if(rank==0) {
 18        std::string image_file = "../../../common/mnist_data/one.raw";
 19        std::ifstream fin(image_file, std::ios::binary);
 20        std::ostringstream ostream;
 21        ostream << fin.rdbuf();
 22        fin.close();
 23
 24        const std::string tmp = ostream.str();
 25        std::memcpy(img.data(), tmp.data(), img.size()*sizeof(float));
 26    }
 27
 28    // Broadcast the image to all MPI ranks.  This is more efficient
 29    // thank all ranks loading the same file.  This is specific
 30    // to this example.
 31    MPI_Bcast(img.data(), 28*28, MPI_FLOAT, 0, MPI_COMM_WORLD);
 32    MPI_Barrier(MPI_COMM_WORLD);
 33
 34    if(rank==0)
 35        std::cout<<"All ranks have MNIST image"<<std::endl;
 36
 37    // Declare keys that we will use in forthcoming client commands
 38    std::string in_key = "mnist_input_rank_" + std::to_string(rank);
 39    std::string script_out_key = "mnist_processed_input_rank_" +
 40                                 std::to_string(rank);
 41    std::string out_key = "mnist_output_rank_" + std::to_string(rank);
 42
 43    // Put the image tensor on the database
 44    client.put_tensor(in_key, img.data(), {1,1,28,28},
 45                      SRTensorTypeFloat, SRMemLayoutContiguous);
 46
 47    // Run the preprocessing script
 48    client.run_script(script_name, "pre_process",
 49                      {in_key}, {script_out_key});
 50
 51    // Run the model
 52    client.run_model(model_name, {script_out_key}, {out_key});
 53
 54    // Get the result of the model
 55    std::vector<float> result(1*10);
 56    client.unpack_tensor(out_key, result.data(), {10},
 57                         SRTensorTypeFloat, SRMemLayoutContiguous);
 58
 59    // Print out the results of the model for Rank 0
 60    if(rank==0)
 61        for(size_t i=0; i<result.size(); i++)
 62            std::cout<<"Rank 0: Result["<<i<<"] = "<<result[i]<<std::endl;
 63
 64    return;
 65}
 66
 67int main(int argc, char* argv[]) {
 68
 69    // Initialize the MPI comm world
 70    MPI_Init(&argc, &argv);
 71
 72    // Retrieve the MPI rank
 73    int rank;
 74    MPI_Comm_rank(MPI_COMM_WORLD, &rank);
 75
 76    // Initialize a Client object
 77    bool cluster_mode = true; // Set to false if not using a clustered database
 78    SmartRedis::Client client(cluster_mode);
 79
 80    // Set the model and script that will be used by all ranks
 81    // from MPI rank 0.
 82    if(rank==0) {
 83        // Build model key, file name, and then set model
 84        // from file using client API
 85        std::string model_key = "mnist_model";
 86        std::string model_file = "../../../"\
 87                                 "common/mnist_data/mnist_cnn.pt";
 88        client.set_model_from_file(model_key, model_file,
 89                                "TORCH", "CPU", 20);
 90
 91        // Build script key, file name, and then set script
 92        // from file using client API
 93        std::string script_key = "mnist_script";
 94        std::string script_file = "../../../common/mnist_data/"
 95                                "data_processing_script.txt";
 96        client.set_script_from_file(script_key, "CPU", script_file);
 97
 98        // Get model and script to illustrate client API
 99        // functionality, but this is not necessary for this example.
100        std::string_view model = client.get_model(model_key);
101        std::string_view script = client.get_script(script_key);
102    }
103
104    // Run the MNIST model
105    MPI_Barrier(MPI_COMM_WORLD);
106    run_mnist("mnist_model", "mnist_script", client);
107
108    if(rank==0)

110
111    // Finalize MPI Comm World
112    MPI_Finalize();
113
114    return 0;
115}

The Fortran API examples are written to connect to a non-cluster Redis database. Update the Client constructor call to connect to a Redis cluster.

Tensors¶

The SmartRedis Fortran client is used to communicate between a Fortran client and the Redis database. In this example, the client will be used to send an array to the database and then unpack the data into another Fortran array.

This example will go step-by-step through the program and then present the entirety of the example code at the end.

Importing and declaring the SmartRedis client

The SmartRedis client must be declared as the derived type client_type imported from the smartredis_client module.

program example
  use smartredis_client, only : client_type

  type(client_type) :: client
end program example

Initializing the SmartRedis client

The SmartRedis client needs to be initialized before it can be used to interact with the database. Within Fortran this is done by calling the type-bound procedure initialize with the input argument .true. if using a clustered database or .false. otherwise.

program example
  use smartredis_client, only : client_type

  type(client_type) :: client

  call client%initialize(.false.) ! Change .true. to false if using a clustered database
end program example

Putting a Fortran array into the database

After the SmartRedis client has been initialized, a Fortran array of any dimension and shape and with a type of either 8, 16, 32, 64 bit integer or 32 or 64-bit real can be put into the database using the type-bound procedure put_tensor. In this example, as a proxy for model-generated data, the array send_array_real_64 will be filled with random numbers and stored in the database using put_tensor. This subroutine requires the user to specify a string used as the ‘key’ (here: send_array) identifying the tensor in the database, the array to be stored, and the shape of the array.

Unpacking an array stored in the database

‘Unpacking’ an array in SmartRedis refers to filling a Fortran array with the values of a tensor stored in the database. The dimensions and type of data of the incoming array and the pre-declared array are checked within the client to ensure that they match. Unpacking requires declaring an array and using the unpack_tensor procedure. This example generates an array of random numbers, puts that into the database, and retrieves the values from the database into a different array.

Parallel (MPI) execution¶

In this example, an MPI program that sets a model, sets a script, executes a script, executes a model, sends a tensor, and receives a tensor is shown. This example illustrates how keys can be prefixed to prevent key collisions across MPI ranks. Note that only one model and script are set, which is shared across all ranks. It is important to note that the Client API calls made in this program are equally applicable to non-MPI programs.

This example will go step-by-step through the program and then present the entirety of the example code at the end.

The MNIST dataset and model typically take images of digits and quantifies how likely that number is to be 0, 1, 2, etc.. For simplicity here, this example instead generates random numbers to represent an image.

Initialization

At the top of the program, the SmartRedis Fortran client (which is coded as a Fortran module) is imported using

use smartredis_client, only : client_type

where client_type is a Fortran derived-type containing the methods used to communicate with the RedisAI database. A particular instance is declared via

type(client_type) :: client

An initializer routine, implemented as a type-bound procedure, must be called before any of the other methods are used:

call client%initialize(.true.)

The only optional argument to the initialize routine is to determine whether the RedisAI database is clustered (i.e. spread over a number of nodes, .true.) or exists as a single instance.

If an individual rank is expected to send only its local data, a separate client must be initialized on every MPI task Furthermore, to avoid the collision of key names when running on multiple MPI tasks, we store the rank of the MPI process which will be used as the suffix for all keys in this example.

On the root MPI task, two additional client methods (set_model_from_file and set_script_from_file) are called. set_model_from_file loads a saved PyTorch model and stores it in the database using the key mnist_model. Similarly, set_script_from_file loads a script that can be used to process data on the database cluster.

if (pe_id == 0) then
  call client%set_model_from_file(model_key, model_file, "TORCH", "CPU")
  call client%set_script_from_file(script_key, "CPU", script_file)
endif

This only needs to be done on the root MPI task because this example assumes that every rank is using the same model. If the model is intended to be rank-specific, a unique identifier (like the MPI rank) must be used.

At this point the initialization of the program is complete: each rank has its own SmartRedis client, initialized a PyTorch model has been loaded and stored into the database with its own identifying key, and a preprocessing script has also been loaded and stored in the database

Performing inference on Fortran data

The run_mnist subroutine coordinates the inference cycle of generating data (i.e. the synthetic MNIST image) from the application and then the use of the client to run a preprocessing script on data within the database and to perform an inference from the AI model. The local variables are declared at the top of the subroutine and are instructive to communicate the expected shape of the inputs to the various client methods.

integer, parameter :: mnist_dim1 = 28
integer, parameter :: mnist_dim2 = 28
integer, parameter :: result_dim1 = 10

The first two integers mnist_dim1 and mnist_dim2 specify the shape of the input data. In the case of the MNIST dataset, it expects a 4D tensor describing a ‘picture’ of a number with dimensions [1,1,28,28] representing a batch size (of one) and a three dimensional array. result_dim1 specifies what the size of the resulting inference will be. In this case, it is a vector of length 10, where each element represents the probability that the data represents a number from 0-9.

The next declaration declare the strings that will be used to define objects representing inputs/outputs from the scripts and inference models.

character(len=255) :: in_key
character(len=255) :: script_out_key
character(len=255) :: out_key

Note that these are standard Fortran strings. However, because the model and scripts may require the use of multiple inputs/outputs, these will need to be converted into a vector of strings.

character(len=255), dimension(1) :: inputs
character(len=255), dimension(1) :: outputs

In this case, only one input and output are expected the vector of strings only need to be one element long. In the case of multiple inputs/outputs, change the dimension attribute of the inputs and outputs accordingly, e.g. for two inputs this code would be character(len=255), dimension(2) :: inputs.

Next, the input and output keys for the model and script are now constructed

in_key = "mnist_input_rank"//trim(key_suffix)
script_out_key = "mnist_processed_input_rank"//trim(key_suffix)
out_key = "mnist_processed_input_rank"//trim(key_suffix)

As mentioned previously, unique identifying keys are constructed by including a suffix based on MPI tasks.

The subroutine, in place of an actual simulation, next generates an array of random numbers and puts this array into the Redis database.

call random_number(array)
call client%put_tensor(in_key, array, shape(array))

The Redis database can now be called to run preprocessing scripts on these data.

inputs(1) = in_key
outputs(1) = script_out_key
call client%run_script(script_name, "pre_process", inputs, outputs)

The call to client%run_script specifies the key used to identify the script loaded during initialization, pre_process is the name of the function to run that is defined in that script, and the inputs/outputs are the vector of keys described previously. In this case, the call to run_script will trigger the RedisAI database to execute pre_process on the generated data (stored using the key mnist_input_rank_XX where XX represents the MPI rank) and storing the result of pre_process in the database as mnist_processed_input_rank_XX. One key aspect to emphasize, is that the calculations are done within the database, not on the application side and the results are not immediately available to the application. The retrieval of data from the database is demonstrated next.

The data have been processed and now we can run the inference model. The setup of the inputs/outputs is the same as before, with the exception that the input to the inference model, is stored using the key mnist_processed_input_rank_XX and the output will stored using the same key.

inputs(1) = script_out_key
outputs(1) = out_key
call client%run_model(model_name, inputs, outputs)

As before the results of running the inference are stored within the database and are not available to the application immediately. However, we can ‘retrieve’ the tensor from the database by using the unpack_tensor method.

call client%unpack_tensor(out_key, result, shape(result))

The result array now contains the outcome of the inference. It is a 10-element array representing the likelihood that the ‘image’ (generated using the random numbers) is one of the numbers [0-9].

Key points

The script, models, and data used here represent the coordination of different software stacks (PyTorch, RedisAI, and Fortran) however the application code is all written in standard Fortran. Any operations that need to be done to communicate with the database and exchange data are opaque to the application.

  2
  3  use mpi
  4  use iso_c_binding
  5  use smartredis_client, only : client_type
  6
  7  implicit none
  8
  9#include "enum_fortran.inc"
 10
 11  character(len=*), parameter :: model_key = "mnist_model"
 12  character(len=*), parameter :: model_file = "../../../common/mnist_data/mnist_cnn.pt"
 13  character(len=*), parameter :: script_key = "mnist_script"
 14  character(len=*), parameter :: script_file = "../../../common/mnist_data/data_processing_script.txt"
 15
 16  type(client_type) :: client
 17  integer :: err_code, pe_id
 18  integer(kind=enum_kind) :: result
 19  character(len=2) :: key_suffix
 20
 21  ! Initialize MPI and get the rank of the processor
 22  call MPI_init(err_code)
 23  call MPI_comm_rank( MPI_COMM_WORLD, pe_id, err_code)
 24
 25  ! Format the suffix for a key as a zero-padded version of the rank
 26  write(key_suffix, "(A,I1.1)") "_",pe_id
 27
 28  ! Initialize a client
 29  result = client%initialize(.true.) ! Change .false. to .true. if not using a clustered database
 30  if (result .ne. SRNoError) stop 'client%initialize failed'
 31
 32  ! Set up model and script for the computation
 33  if (pe_id == 0) then
 34    result = client%set_model_from_file(model_key, model_file, "TORCH", "CPU")
 35    if (result .ne. SRNoError) stop 'client%set_model_from_file failed'
 36    result = client%set_script_from_file(script_key, "CPU", script_file)
 37    if (result .ne. SRNoError) stop 'client%set_script_from_file failed'
 38  endif
 39
 40  ! Get all PEs lined up
 41  call MPI_barrier(MPI_COMM_WORLD, err_code)
 42
 43  ! Run the main computation
 44  call run_mnist(client, key_suffix, model_key, script_key)
 45
 46  ! Shut down MPI
 47  call MPI_finalize(err_code)
 48
 49  ! Check final result
 50  if (pe_id == 0) then
 51    print *, "SmartRedis Fortran MPI MNIST example finished without errors."
 52  endif
 53
